rustup인가 Homebrew인가#

Rust를 설치하기 위해서는, rustup이라는 Rust의 셋업 명령어를 사용하는 것이 추천된다. Homebrew에서 rustc(rust 컴파일러)를 설치하는 것도 가능하지만, 개발에 필요한 각종 커맨드(linter의 clippy등)은 Homebrew가 없기 때문에, 그것들을 설치하는 데 막대한 노동이 필요하기 때문에 추천하지 않는다.

rustup의 설치#

먼저, rustup을 설치한다.
rustup의 설치 방법에는, Homebrew를 사용하는 방법과, 셋업 명령어를 사용하는 방법이 있다.
어떤 방식이든 별로 차이가 없지만, Homebrew가 이미 설치되어 있다면 Homebrew로 rustup을 편하게 설치할 수 있다.

  • Homebrew로 설치하는 방법
% brew install rustup-init
% rustup-init
  • 설치 커맨드를 사용하는 경우
% curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

참고로 필자는 후자를 이용하였다.

rust를 사용해보자#

cargo를 사용하여 샘플 프로젝트를 다운로드 할 수 있다.

% cargo new hello_rust
     Created binary (application) `hello_rust` package

다운로드한 프로젝트의 디렉토리로 이동하자.

cd hello_rust

프로젝트를 컴파일하여 실행하자.

% cargo run
   Compiling hello_rust v0.1.0 (/Users/sakura/Downloads/hello_rust)
    Finished dev [unoptimized + debuginfo] target(s) in 0.35s
     Running `target/debug/hello_rust`
Hello, world!

가장 마지막에서 볼 수 있는 것처럼 Hello, world!가 출력되면 성공이다.

Visual Studio Code에서 사용하기 위한 방법#

Visual Studio Code의 확장기의 rust-analyzer를 설치한다.

먼저, rust-analyzer에 필요한 명령어를 rustup 명령어로 설치한다.

% rustup component add rls rust-src rust-analysis
info: downloading component 'rls'
info: installing component 'rls'
info: downloading component 'rust-src'
info: installing component 'rust-src'
info: downloading component 'rust-analysis'
info: installing component 'rust-analysis'

다음으로, Visual Studio Code에서, rust-analyzer의 확장기를 설치한다.

이걸로 Visual Studio Code에서 rust를 사용하기 위한 준비는 끝이다.

아까 cargo로 다운로드한 hello_rust의 프로젝트를 Visual Studio Code로 열어보자.

src라는 디렉초리 안에 main.rs라는 파일이 있다. 이것이 rust의 소스코드 파일이다. 이 파일을 열면 syntax별로 색깔이 보이는 것을 알 수 있다.
